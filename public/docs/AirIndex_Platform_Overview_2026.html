What AirIndex Is

AirIndex is the market intelligence platform for advanced air mobility. We track readiness conditions across 21+ U.S. metropolitan areas — scoring legislation, infrastructure, operator activity, and regulatory posture against a published 7-factor methodology updated continuously from primary federal and state sources.

Built for operators, infrastructure developers, government agencies, insurance underwriters, and investors making capital allocation decisions in AAM. The platform serves as both a monitoring tool (scores, trajectories, alerts) and an advisory layer (gap analysis, scenario modeling, precedent lookups).

AirIndex is not a forecast or demand model. It measures structural readiness — the infrastructure, regulatory framework, and operator engagement that must be in place before commercial eVTOL operations can begin.

04

Intelligence Architecture

Container 1: Factor Knowledge Base (FKB)
Live

Authoritative source of truth for scoring factors. 8 factor codes (7 active + 1 retired), versioned weight configuration, 21 data sources with reliability ratings, 147 per-market factor scores with confidence levels, analyst override audit trail. Scoring engine reads weights from FKB at runtime — methodology changes are database updates, not code deployments.

Container 2: Market Context Store (MCS)
Foundation

Market registry with 23 cities. Peer market mappings, state-level context. Foundation table referenced by all other containers.

Container 3: Operator Intelligence Database (OID)
Planned

6-stage operator deployment model (ANNOUNCED → COMMERCIAL_OPS). 11+ operators tracked. Event taxonomy with auto-advance staging. Feeds OPR factor scoring with confidence-weighted operator sub-scores.

Container 4: Regulatory Precedent Library (RPL)
Live

1,912 classified regulatory documents. 15 document types, 4 momentum directions (POS/NEG/NEU/MIX), 3 significance levels. Factor mappings link every document to affected scoring factors. City associations link documents to affected markets. 406 state legislation records with jurisdiction, bill number, chamber, and legislative stage. Precedent lookup engine answers: “What did Dallas do to reach ADVANCED?”

Container 5: Federal Programs Intelligence Store (FPIS)
Planned

10 federal programs cataloged. Dual consumer: Gap Analysis (client-facing) + VDG Grant Tracker (internal). City + factor gap → matching federal programs with peer precedents.

  • Regulations.gov API (daily, FAA dockets)

Pipeline Stats
  • 1,912 ingested records
  • 948 AI classifications (Claude Haiku)
  • ~80% classification accuracy (v4 prompt)
  • 5 prompt versions tracked with commit hashes
  • Daily ingestion at 06:00 UTC
  • Snapshot after ingestion, auto-review after snapshot
  • Conservative auto-reviewer: drops false signals, doesn’t promote false ones
